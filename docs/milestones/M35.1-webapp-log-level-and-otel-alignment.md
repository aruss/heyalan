# Milestone M35.1: Remove Pino and Send WebApp Logs Directly to OTEL

## Summary
Replace the WebApp server logger with a small server-only logger built on `@opentelemetry` logs packages. Remove `pino` and `pino-opentelemetry-transport` completely from the WebApp codebase and dependency graph, send logs directly to the OTEL collector over gRPC, and make both configuration and logger method names follow `.NET` terminology.

This milestone is a focused follow-on to M35. It replaces the earlier Pino-based logging direction for the WebApp and standardizes the final design on a custom OTEL-backed logger.

## Goals
- [x] Remove all `pino` packages from `BuyAlan.WebApp`.
- [x] Remove all `pino` references from `BuyAlan.WebApp`.
- [x] Replace the current logger with a minimal server-only OTEL logger.
- [x] Standardize `LOG_LEVEL` and logger method names on `.NET` terminology.
- [x] Keep startup logging, health probe logging, and fatal-path flushing working with the new logger.
- [x] Preserve shared OTEL resource metadata across logs, traces, and metrics.

## Non-Goals
- [ ] No browser/client log forwarding.
- [ ] No WebApi logging behavior changes.
- [ ] No OTEL infrastructure or Aspire env wiring changes.
- [ ] No schema, migration, OpenAPI, or generated file changes.
