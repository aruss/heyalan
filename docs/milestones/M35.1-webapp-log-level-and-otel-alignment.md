# Milestone M35.1: WebApp Log Level and OTEL Resource Alignment

## Summary
Finish the WebApp logging alignment work on top of M35 so one `LOG_LEVEL` value can be shared across WebApi and WebApp, the startup debug event contains safe environment context, and OTEL resource metadata is consistent across logs, traces, and metrics.

This milestone is a focused follow-on to M35. It does not replace the shared logger foundation, process-level error capture, or OTLP log export already added there.

## Scope
- [x] Normalize WebApp `LOG_LEVEL` handling to accept .NET-style level names.
- [x] Preserve backward compatibility for existing Pino-style level names.
- [x] Enrich the startup debug event with safe environment metadata.
- [x] Add a request-triggered OTEL log probe on the existing health route.
- [x] Align OTEL resource attributes between log export and the Node SDK setup.
- [x] Add focused validation for log-level mapping and startup metadata payloads.
- [ ] End-to-end runtime verification against stdout and the collector.

## Non-Goals
- [ ] No browser/client log forwarding.
- [ ] No WebApi logging behavior changes.
- [ ] No OTEL infrastructure or container environment changes.
- [ ] No schema, migration, OpenAPI, or generated file changes.
