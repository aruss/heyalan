# Milestone M14.1: Onboarding Behavour Refactoring

## Goal
Allow users to skip Square connect during onboarding so they can explore the admin dashboard before activation, while keeping Square required to complete onboarding and mark the user as onboarded.

## Scope
- **Backend (`HeyAlan`)**
  - Refactor onboarding step progression so missing Square always anchors onboarding to `square_connect` on re-entry.
  - Keep final activation requirements unchanged (`square_connect`, `profile`, `channels`, `invitations` required for finalize).
- **Frontend (`HeyAlan.WebApp`)**
  - Keep skip UX in onboarding step 1.
  - Ensure user guidance text clearly states setup can be postponed and resumed later from dashboard.
  - Persist and prefill agent profile draft (`name`, `personality`) when onboarding is reopened.
  - Do not display `Invite Team` when Square is not connected.
  - Route users to `/admin` when they reach post-channel steps without Square instead of calling blocked invitation/finalize APIs.
- **Tests (`HeyAlan.Tests`)**
  - Add/adjust onboarding service tests for progression semantics and finalize guard behavior.

## Non-Goals
- Changing onboarding endpoint contracts.
- Making Square optional for final onboarding completion.
- Changing admin route-guard behavior.
- Editing generated API client files.

## Architecture Decisions (Locked)
- [x] Skip behavior is backend-driven (not frontend-only state).
- [x] Square remains required for operational activation (`isOnboarded=true`).
- [x] Existing onboarding API contracts remain stable.

- [x] Keep `canFinalize` requirements unchanged.
