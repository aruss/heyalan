# Milestone M23.1: Square Catalog Description Exposure and Inventory Table Polish

## Summary
Expose the already-synced Square product description through the subscription catalog diagnostics API and use it in the inventory settings page so operators see more useful product context.

This milestone is a follow-on to M23. It does not change Square sync cadence, webhook behavior, storage shape, assignment semantics, or runtime message-processing logic. The focus is narrow:
- add cached `Description` to the subscription catalog diagnostics API contract,
- regenerate the WebApp client after the contract change,
- simplify the inventory settings catalog snapshot table by removing locations and `squareVariationId`,
- show description beneath the product name using smaller secondary text.

## Dependencies and Current Codebase Baseline
- [x] `SubscriptionCatalogProduct.Description` already exists in the data model and is populated by the Square sync pipeline.
- [x] Internal catalog read services already expose description in `SubscriptionCatalogProductResult`.
- [x] `GET /subscriptions/{subscriptionId}/square/catalog/products` currently omits description from `SubscriptionSquareCatalogProductItem`.
- [x] The inventory settings page in `BuyAlan.WebApp/src/app/admin/settings/inventory/page.tsx` currently renders a cached catalog snapshot table that still shows `squareVariationId` inline and a separate `Locations` column.
- [x] The agent inventory settings page remains a placeholder, so there is no second implemented catalog-listing UI to update in this milestone.

## User Decisions (Locked)
- [x] Description must be exposed through the subscription catalog diagnostics API, not derived client-side.
- [x] The inventory settings page must show description in the product cell where `squareVariationId` is currently rendered.
- [x] Description must use smaller secondary text styling.
- [x] The inventory settings table must remove the `Locations` column.
- [x] The inventory settings table must remove visible `squareVariationId`.
- [x] Scope is "all current catalog UIs," which currently resolves to the inventory settings page only because the agent inventory page is still unimplemented.
